## 0. P1의 목적

P0가 만든 환경(primary sim = **DLO-Lab**, `DLOLabEnv`) 위에 **HACMan-style black-box contact critic baseline**을 구축하고 T1–T2 task에서 안정적으로 학습시킨다. 이 baseline은 (1) P2 probing의 **진단 대상**, (2) P3 구조 비교의 **주 대조군**이다. 따라서 P1의 품질 기준은 "SOTA 성능"이 아니라 **재현 가능하고 진단 가능한, 비자명하게 학습된 critic**이다.

P1이 만들지 않는 것 (P2+ 범위 — 구현 금지): probe 실험 일체, DGCC variants (V1/V2/V3), f_resp/response head, GreedyResp, matched-dim latent, OOD 평가 sweep, 실로봇 관련 일체.

## 5. Task Suite 정의 (M0 구현 대상)

공통: rope 기본 도메인 = length 1.0 m, n_segments 32, bend/twist/friction 1.0, radius 0.005 (P0 수집 기본과 동일). init_shape 4종 {straight, u_bend, s_curve, random_smooth} 균등. **OOD 도메인 사용 금지 (P6).**

```
T1 (local shaping, 3 tasks — goal이 단순·국소적):
  T1-a straighten:          goal = 직선 template + anchor(현재 centroid 유지)
  T1-b single_bend:         goal = U 계열 단일 곡률 template (u_bend 파라미터 1개 변형)
  T1-c endpoint_reposition: goal = 현재 shape 유지 + anchor를 [0.2, 0.4] m 범위 이동
T2 (global shape matching):
  절차적 goal 생성기: {S, U, L, zigzag, smooth-random} family, 파라미터 무작위화
  분할: train 500 / validation 50 / held-out eval 100 (seed 고정, 분할 파일 커밋)
  비대칭 goal 포함 (좌우 비대칭 곡률 — 잘못된 contact 선택 시 1-step 도달 불가)
```

Episode: 최대 **T=10** primitives (연구 계획 §6.1 값 — M0에서 채택, 조정 시 기록), 성공(D < ε_succ) 시 조기 종료. 에피소드 내 재파지 허용 (primitive 정의상 자동).

Reward (전역 규칙 4의 상수, D는 불변 metric):

```
r_t = α [D(X_t, G) − D(X_{t+1}, G)] − c_step + 1{D(X_{t+1}, G) < ε_succ} · R_succ
D = 길이 정규화 correspondence L2 (orientation canonicalization 포함, P0 §8 개정판 그대로 재사용)
성공 판정과 reward가 동일 D를 사용 — 코드 경로도 P0의 goals/distance.py를 재사용 (재구현 금지)
```

## 6. Baseline Architecture (M1 구현 대상 — 연구 계획 §6.7 시작값)

```
입력 node feature: (x_i ∈ R³, 호길이 좌표 σ_i, goal 대응점 잔차 g_i − x_i)   # 대응은 canonical flip 기준
encoder: shared MLP(64) → 1D CNN 3층 (kernel 5, dilation 1/2/4, d=128) + global max-pool broadcast
         → per-node embedding h_i ∈ R^256 (local 128 + global 128 concat)
critic (twin): per-point MLP(256, 256): [h_p, u] → Q_i(s, g, p, u)   # goal은 encoder 입력으로 조건화
actor: per-point MLP(256, 128): h_i → u_i = (Δ ∈ R³ tanh·0.15, lift logit)
파라미터 총량 ~1–2M. 이 구조가 P2 probe·P3 variants의 공통 backbone이 되므로
encoder / critic-trunk / head 경계를 모듈로 분리해 둘 것 (latent hook 지점).
```

## 7. 학습 알고리즘 (M1–M2 구현 대상)

```
TD3-style off-policy:
  y = r + γ · min_i Q_target_i(s', g, p'*, ũ')      γ = 0.95
  p'* 선택·평가 분리 (double-Q decoupling — 연구 계획 §6.6):
    p'* = argmax_p Q_target_1(s', g, p, u_target(s', p))
    평가는 min(Q_target_1, Q_target_2) at p'*
  ũ' = u_target(s', p'*) + clip(N(0, 0.05), ±0.1)   # target policy smoothing (u 스케일 기준)
L_actor = − E_s [ (1/K) Σ_p Q_min(s, g, p, u_θ(s, p)) ]   # 전-후보 갱신 (P0.md와 동일 정식화)
탐험: ε-greedy over p (1.0 → 0.1, 학습 예산의 첫 30%에서 선형) + Gaussian on u (σ = 0.03 m)
기본값: lr 3e-4 (Adam) · batch 256 · replay 5×10⁵ · UTD 1 · warmup 5k random transitions
       · τ = 0.005 · grad-norm clip 10 · HER 미사용 (M4 조건부)
평가: deterministic (argmax p, noise-free u), 수집 25k transitions마다
      T1: task당 100 episodes · T2: validation 50 goals × 2 episodes
체크포인트: 25k마다 저장 + best-on-validation 별도 유지 (eval goal에 비접근 — leakage 차단)
학습 예산 시작값: T1 run당 1×10⁵ transitions · T2 run당 3×10⁵ transitions · seed 3개씩
```

수집 처리량 전제 (P0 실측): n_envs=64에서 3.61 transitions/s → 3×10⁵ ≈ 23 h. **M0의 처리량 프로브 결과로 n_envs를 정하고 예산 소요를 보고할 것** (P0은 16→64 env 스케일링에서 거의 선형이었음: 0.91/1.82/3.61 tr/s).
